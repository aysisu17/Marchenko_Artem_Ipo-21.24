ls (list) - список файлов и папок
cd (changedirectory) - смена папки
pwd (print working directory) - показывает текущую директорию
git init - эта команда инициализирует репозиторий в текущей папке (появляется скрытый .git)
`ls -a` - показывает также скрытые файлы в папке
git status - эта команда показывает, в каком состоянии в данный момент находится наш репозиторий.
git add - передать файл под контроль Git. (git add название-файла или git add `*/.`).
git commit -m "название коммита" - фиксация изменений в репозитории.
git rm FILE – удаляет файл из индекса и из рабочей папки. (требует фиксации)
git rm --cachedFILE – удаляет файл только из индекса Git. (требует фиксации)
git log - история коммитов
git reset HEAD– откат к последнему коммиту (здесь HEAD–указатель на текущий коммит).
git reset --hard HEAD - самая мощная и самая опасная команда – откат к указанному коммиту с потерей всех изменений!
git checkout название-файла - вернуться в состояние до изменений.
git push - эта команда берет имеющиеся локальные изменения и отправляет их в удалённый репозиторий.
git pull -эта команда загрузит изменения из удалённого репозитория.
cat `полное_название` - посмотреть содержимое файла.
git remote add - добавляет к текущему репозиторию ссылку на удалённый под каким-то именем.
git remote - покажет список прикрепленных к нашему репозиторию удалённых репозиториев.
git fetch - берет информацию обо всех изменениях в удалённом репозитории и переносит в текущий.
git checkout --track `название репозитория / ветвь` - создать связь текущей ветки данного репозитория с ветвью master репозитория.
git pull – команда, которая получает изменения.
git tag– это команда, которая выводит состояние меток в текущем репозитории.
git show - увидеть помеченный коммит.
git tag -1 '1.' - вывод меток, начинающихся на 1.
git tag -d 1.0 - удаление метки.
git push `название удал репоз` 1.0 - передача метки на удаленный репозиторий.
git push `название удал репоз` --tags - для всех меток 
mkdir - созданиe новых папок (директорий) в файловой системе перед добавлением их в репозиторий.
git branch `название ветки` - создание новой ветки
git checkout `название ветки` - переключиться на новую ветку
git merge - слияние двух веток
git branch –v - полный перечень веток репозитория
##### Состояния файлов в git:
1) Untracked – файл просто находится в папке, но для Git он неизвестен, его в истории нет и никогда не было.
2) Staged – подготовленный для фиксации изменений (командой add).
3) Unmodified–означает, что файл со времени последней фиксации не менялся.
4) Modified – измененный файл.
